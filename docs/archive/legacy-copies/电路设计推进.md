## 解决过程及方案：

以下是现用忆阻器的lib文件定义：

```
* PSpice Model Editor - Version 16.5.0
*$
.SUBCKT memristor_V Plus Minus PARAMS:
+ Ron=10 Roff=420 Rinit=20 D=3N uv=1.6E-12 p=10 Vt1=0.19V Vt2=-0.37V Ioff=1e-5 I0=1e-3
Gx 0 x value={(Ioff/(I(Emem)-I0)*uv*(stp(V(plus,minus)-Vt1))*Ron/D^2+I(Emem)*uv*stp(Vt2-V(plus,minus))*Ron/D^2)*f(V(x),p)}
Cx x 0 1 IC={(Roff-Rinit)/(Roff-Ron)}
Raux x 0 1T
Emem plus aux value={-I(Emem)*V(x)*(Roff-Ron)}
Roff aux minus {Roff}
Eflux flux 0 value={SDT(V(plus,minus))}
Echarge charge 0 value={SDT(I(Emem))}
.func f(x,p)={1-(2*x-1)^(2*p)}
*proposed window function
.ENDS memristor_V
*$
```

问题出现在其使用的窗函数上。 f(x,p)={1-(2*x-1)^(2*p)}使得当x=1或x=0时，函数计算结果为0

该模型中，x值由受控源Gx的输入控制，而Gx的表达式=（一些函数）*f(V(x),p)}

当x=1或x=0，Gx恒=0，x不再变化。

虽然biolek窗函数的变化斜率在实际的连续过程中不会使得x变为0或1的值，但在模拟过程中确实会出现这样的情况，需要寻找解决的方案。

观察到，Gx=[(正向电压，阻值减小，x减小情况)+(逆向电压，阻值增大，x增大情况)]*f(V(x),p)

无论x等于1还是等于0，都既不允许增大，又不允许减小。

然而我们只是希望限制x在(0,1)范围内，实际上x=1时可以让他减小，x=0时可以增大

### 改进方案：Gx=[(正向电压，阻值减小，x减小情况)xf1(V(x),p)+(逆向电压，阻值增大，x增大情况*)xf2(V(x),p)]

其中，两个功能函数分别实现一半的窗函数功能。实际代码如下：

```
.SUBCKT memristor_V Plus Minus PARAMS:
+ Ron=10 Roff=420 Rinit=20 D=3N uv=1.6E-12 p=10 Vt1=0.19V Vt2=-0.37V Ioff=1e-5 I0=1e-3
Gx 0 x value={ Ioff/(I(Emem)-I0)*uv*stp(V(Plus,Minus)-Vt1)*Ron/D^2*f_inc(V(x),p) + I(Emem)*uv*stp(Vt2-V(Plus,Minus))*Ron/D^2*f_dec(V(x),p)}
Cx x 0 1 IC={(Roff-Rinit)/(Roff-Ron)}
Raux x 0 1T
Emem Plus aux value={-I(Emem)*V(x)*(Roff-Ron)}
Roff aux Minus {Roff}
Eflux flux 0 value={SDT(V(Plus,Minus))}
Echarge charge 0 value={SDT(I(Emem))}
.func f_inc(x,p)={1-x^(2*p)}
.func f_dec(x,p)={1-(1-x)^(2*p)}
.ENDS memristor_V

```

这是biolek窗函数在原文中的定义：![image-20260122223042629](C:\Users\lenovo\AppData\Roaming\Typora\typora-user-images\image-20260122223042629.png)

实际上，窗函数2p次方内的常数项并不恒为-1。![image-20260122223618394](C:\Users\lenovo\AppData\Roaming\Typora\typora-user-images\image-20260122223618394.png)

通过我的区分，使函数在正负电流有不同的表达式，反而契合biolek窗函数的真实情况

更改忆阻代码后，忆阻器件能够正常工作，并且同时解决了先前无法将初始值设定为ron/roff的问题。
